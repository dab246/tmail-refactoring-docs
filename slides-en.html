Analysis

2 Core Technical Problems

⚠ Problem A — Shared State

"Where does app state live?"

  • 42 Rx fields in MailboxDashBoardController
  • 5 controllers depend on a single God Object
  • God Object can't be removed — everyone needs it
  • Tests require mocking the entire Dashboard (40+ lines setUp)

⚠ Problem B — Inter-Controller Comm

"How does controller A notify B?"

  • ever(dashboard.dashBoardAction, callback)
  • Callback closures 85 – 159 lines
  • Not type-safe: Rxn<UIAction>
  • Every controller receives noise from unrelated actions

These two problems are coupled — you can't fully solve A without solving B, and vice versa.

Pillar 2 — AppEventBus

AppEventBus: Type-safe, Zero Coupling

class AppEventBus extends GetxService {
  final _ctrl =
    StreamController<AppEvent>.broadcast();

  Stream<AppEvent> get stream => _ctrl.stream;

  void dispatch(AppEvent event) {
    if (!_ctrl.isClosed) _ctrl.add(event);
  }

  @override
  void onClose() {
    _ctrl.close();     // cleanup on logout
    super.onClose();
  }
}

// Sealed class — compile-time exhaustive
sealed class AppEvent {}

final class MailboxSelectedEvent
    extends AppEvent {
  final PresentationMailbox mailbox;
  MailboxSelectedEvent(this.mailbox);
}

final class EmailDeletedEvent
    extends AppEvent {
  final List<EmailId> emailIds;
  EmailDeletedEvent(this.emailIds);
}
// Subscriber — receives only what it needs
class ThreadController
    extends BaseController
    with EventBusSubscriberMixin {

  @override
  void onInit() {
    super.onInit();
    // Only receives MailboxSelectedEvent
    _eventBus.stream
      .whereType<MailboxSelectedEvent>()
      .listen(_onMailboxSelected)
      .addTo(compositeSubscription);
      // ↑ auto-cancelled on dispose
  }
}
BENEFITS
  • Zero coupling — controllers don't know each other
  • Type-safe at compile time
  • New event type → no changes to existing code (OCP)
  • Auto cleanup — no leaks
  • Easy to test: inject mock stream

Rules

8 Rules Applied Across the Codebase

✅ DOs

  • Constructor injection only — no Get.find() in business logic
  • Each controller ≤ 300 lines
  • Each constructor ≤ 8 dependencies
  • Each controller ≤ 10 Rx fields
  • EventBus for cross-feature events only
  • Subscribe using whereType<T>()
  • Auto cleanup: .addTo(compositeSubscription)
  • Domain Contracts — don't import implementations

❌ DON'Ts

  • No more Rxn<UIAction> observables
  • No more Get.find<T>() in field declarations
  • Don't dispatch an event inside a handler of the same type (loop)
  • Don't subscribe to the full stream then if-else
  • Don't create a GetxService holding all state (God Service)
  • Don't use EventBus for UI events (button tap, scroll)
  • Don't manually cast events (as MyEvent)
  • Don't hold a StreamSubscription without cancelling it
